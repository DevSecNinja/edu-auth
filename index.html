2 · Admins: the non-negotiable gold lane

For people with admin rights there is no "in-between." These four controls go together. Anything less leaves the whole tenant exposed.

The four admin essentials

If a teacher or student account is breached, the blast radius is one mailbox. If an admin account is breached, it's the whole organisation.

Phishing-resistant MFA

FIDO2 key or passkey only — no SMS, no push that can be tricked or fatigued.

Just-in-time roles (PIM)

Admin rights are switched on for a task and a time window, then switched off automatically.

Separate cloud-only account

A dedicated admin identity that never reads email or browses the web — kept away from daily risk.

Privileged Access Workstation (optional)

A clean, locked-down device used only for admin work. The gold standard for the highest-risk roles.

3 · "Students can't use a phone" — here's what to do instead

Phones are one way to do strong sign-in, not the only way. These options are device-bound and phishing-resistant, and none of them need the student to own or carry a phone.

The phone-free menu

Device situationRecommended method
1:1 Windows deviceWindows Hello for Business face / PIN
1:1 MacPlatform SSO + Secure Enclave Touch ID
1:1 iPad / Android (if allowed)Passkey in Authenticator
Shared Windows / MacFIDO2 security key recommended for shared

First-time setup for all of these uses a Temporary Access Pass (TAP) — a short code from IT. No phone needed to get started.

Make TAP painless for IT

The real objection is usually admin workload, not technology. These cut the effort dramatically:

  • TAP-only, no password — issue a TAP instead of a first password, so there's no extra step.
  • Hand out at enrolment — generate the TAP when a device is given to the student / during Autopilot.
  • Delegate to teachers or local IT — a scoped role lets trusted staff issue TAPs without full admin rights.
  • Self-service trigger — student only calls IT for a TAP at first setup.
  • Multi-use TAP with a sensible lifetime — fewer reissues across the rollout.
  • Watch the reports — Authentication Methods Activity shows rollout progress.
Conditional Access ties it together: require a compliant device and phishing-resistant strength for student apps. For shared devices, FIDO2 keys are the Microsoft-recommended method.
